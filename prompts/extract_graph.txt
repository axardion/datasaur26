-Goal-
You are extracting a clinical knowledge graph from Kazakhstan clinical protocol text (МКБ-10 / ICD-10). Identify all clinically relevant entities and relationships for diagnosis assistance: diagnoses, symptoms/complaints, ICD-10 codes, and protocols.

-Steps-
1. Identify all entities. For each identified entity, extract:
- entity_name: Short name, capitalized (e.g. diagnosis name, symptom phrase, ICD code as written, protocol title)
- entity_type: One of: [{entity_types}]
- entity_description: Brief clinical description (symptoms covered, ICD meaning, or what the protocol is for)
Format each entity as ("entity"<|><entity_name><|><entity_type><|><entity_description>)

2. From the entities in step 1, identify pairs that are clinically related. For each pair extract:
- source_entity, target_entity: names from step 1
- relationship_description: why they are related (e.g. "symptom suggests diagnosis", "protocol covers ICD code", "diagnosis presents with symptom")
- relationship_strength: 1-10 (10 = very strong clinical link)
Format each as ("relationship"<|><source_entity><|><target_entity><|><relationship_description><|><relationship_strength>)

3. Return output in English as a single list. Use **##** as the list delimiter.

4. When finished, output <|COMPLETE|>
